# Prompt Examples

下面这些 prompt 可以直接复制到 Codex 里使用。

## 单条要理问答生成一张图

```text
Use $lizhi-illustrations 为下面这条要理问答生成一张 16:9 解析配图。
画面要怪诞但清爽，Lizhi 必须承担核心动作。
中文标注最多 5 个，短一点。

问：<粘贴问题>
答：<粘贴回答>
```

## 单个神学概念生成一张图

```text
Use $lizhi-illustrations 为“<概念或命题>”生成一张 16:9 解析配图。
先想清楚这张图讲的是定义、对比、进程、关系还是张力，再发明一个低科技隐喻。
Lizhi 必须承担核心动作，不要画成课件。
```

## 只做配图规划

```text
Use $lizhi-illustrations 先不要生图。
请分析下面这段文本哪里值得配图，输出 shot list。
每张图写清楚：
- 对应哪条问答、哪个概念，或放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- Lizhi 在图里做什么
- 建议元素
- 建议中文标注词

<粘贴文本>
```

## 一段论述配图

```text
Use $lizhi-illustrations 把下面这段论述生成 4 张 Lizhi 怪诞解析配图。
要求：16:9 横版、纯白背景、黑色手绘线稿、少量红橙蓝中文手写批注。
每张图只讲一个核心结构，不要做 PPT 信息图，不要可爱卡通。

<粘贴文本>
```

## 概念对比图

```text
Use $lizhi-illustrations 为“<概念A>与<概念B>的区别”生成一张对比图。
不要简单画左右两块，请发明一个能承载这组区分的物理隐喻。
Lizhi 在界线处承担区分或连接的动作。
```

## 进程/次序图

```text
Use $lizhi-illustrations 为“<一个有先后次序的进程>”生成一张进程图。
一条主线 3-5 个节点，橙色箭头表达方向，不要把每个节点都解释清楚。
Lizhi 在关键一环承担动作。
```

## 改图：去掉标题

```text
Use $lizhi-illustrations 帮我编辑这张图。
去掉左上角的“定义图 / 流程图”标题和下划线，其他内容保持不变。
不要新增任何文字或物件。
```

## 改图：增强 Lizhi 参与感

```text
Use $lizhi-illustrations 这张图方向对，但 Lizhi 有点像装饰。
请保持核心意思不变，重生成一版：让 Lizhi 成为真正推动结构运转的人。
画面更怪一点，但仍然纯白、清爽、少字。
```
